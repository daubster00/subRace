## 프로젝트 개요
- **이름**: SubRace / 라이브서브랭크
- **위치**: `D:\projects\SubRace`
- **목적**: 일본 유튜브 채널의 구독자 수 순위를 실시간에 가깝게 보여주는 단일 페이지 대시보드. 운영자가 OBS로 캡처해 방송에 송출하는 용도.
- **핵심 기능 요약**:
  - 일본 유튜브 상위 150개 채널을 백그라운드로 추적
  - 화면에는 TOP 50만 표시 (또는 100, 확정 필요 — 아래 *불일치 사항* 참조)
  - 구독자 수 변동을 디지털 카운터 롤링 애니메이션으로 표현
  - 순위 변동/진입/이탈을 모션으로 자연스럽게 표시
  - 두 외부 API를 서로 다른 주기로 폴링 (Social Blade 48h, YouTube Data API 6h)

- **상단 요약 정보 (3장 카드)**: 시청 중 / 좋아요 / 업데이트

- **하단 상태 패널 3종**: 실시간 상태 / 다음 업데이트 예정 / 순위 변동 알림
- **기준 시각 표시**: `2025.05.24 14:30:00 기준 (한국시간)` 형식 타임스탬프

## 동작 사양 (브리프 문서 §3~§10 요지)

### 데이터 운영 범위
- 화면: 항상 TOP 50만 표시
- 백그라운드: 상위 150개 채널 데이터 보유
- API 업데이트 시 150개 전체 재정렬 → 상위 50개만 노출
- 50위권 진입/이탈은 갑작스러운 교체 X, 모션으로 자연스럽게

### 애니메이션 3종
1. **TOP 50 내부 순위 변동**: 카드가 부드럽게 이동, 순위 변경 카드에 짧은 강조 효과
2. **50위권 진입**: 페이드 인 / 아래→위 슬라이드 / 하이라이트 중 택, 짧은 시간 강조 유지
3. **50위권 이탈**: 페이드 아웃 / 아래로 밀려남 / 흐려짐 중 택

### 구독자 숫자 롤링 카운터
- 단순 증감 보간 X
- **오른쪽 끝 한 자리 숫자부터 촤라락 넘어가며** 왼쪽으로 자릿수별 순차 변화
- 디지털 카운터/전광판 느낌
- 콤마 유지, 자릿수 변화 시 레이아웃 흔들림 없어야 함
- 증가/감소 방향에 따라 초록/빨강 상태 표시 연동

### 구독자 증감 표시
- 카드 오른쪽에 변화량 표시
- 증가: 초록색 상승 / 감소: 빨간색 하락

### 순위 변동 임박 표시
- 두 채널 구독자 수가 매우 가까워 변동 가능성 클 때
- **인접한 두 채널 카드를 동시에** 빨간색 계열로 강조
- 배경 / 테두리 / 은은한 빛 효과 (pulse / glow) — 문구 강조 X

### 85% 추정 알고리즘 (브리프 §10)
- 화면 추정치는 **실제 API 값의 증가 속도를 그대로 따라가지 않음**
- 약 **85% 수준**까지만 자연스럽게 따라가고
- 이후에는 실제 API 표시값을 추월하지 않도록 속도를 늦춤
- 예: API 1,000만 → 1,010만 흐름이면 화면은 약 1,008.5만 수준까지 따라가다 감속
- API 업데이트 시 실제 값과 표시값의 차이를 보정

### 데이터 소스 / 폴링 주기 (브리프 §11)
| 소스 | 역할 | 주기 |
|------|------|------|
| Social Blade | 일본 상위 150개 채널 **목록** 관리 (신규 진입/이탈 후보) | **48시간**마다 1회 |
| YouTube Data API v3 | 그 150개 채널의 **구독자 수 변동** 감지 | **6시간**마다 1회 |
| 프론트 | 갱신된 150개 중 상위 50개만 표시 | — |

## 자료 분석 메모 (생성기가 추출한 키워드/제약/주의점)

### 자료에서 직접 가져온 핵심 제약
- 화면 표시는 50개, 데이터 보유는 150개 — **표시 ≠ 운영 데이터**
- 두 API 주기 차이 (48h vs 6h) — Social Blade는 "목록", YouTube는 "수치"
- 85% 추정 — 실제 값 추월 절대 금지
- 숫자 롤링은 자릿수별 순차, 통째 보간 아님
- 순위 변동 임박은 인접 두 카드 **동시** 강조
- 운영자 전용 / OBS 캡처 — 모바일/멀티 디바이스 고려 X (확정 필요)

### 재사용 가능한 자산
- `자료/live_subrank_coding/` — 디자인 시안에 가까운 작동 프로토타입 (HTML/CSS/JS). Next.js 컴포넌트로 이식 가능성 높음.
- `자료/SubRace_design_50.png`, `_100.png` — 픽셀 참조 시안.

## 확정 사항 (2026-05-07 사용자 확인)
> 브리프 작성 후 사용자와 합의한 결정사항. PRD 작성의 입력으로 사용한다.

1. **표시 범위**: **초기 TOP 50** 노출. 향후 TOP 100으로 확장 가능성 있음 (고객 미확정).
   - **요구사항**: 표시 개수를 **설정값/상수로 추출**하여 50 → 100 전환을 코드 한 곳 변경으로 가능하게 설계.
   - 5개 컬럼(50) / 10개 컬럼(100) 모두 1920×1080에서 가독성 유지되도록 레이아웃 설계.
   - 백그라운드 데이터는 항상 150개 유지(변경 없음).
2. **인증**: **Basic Auth (ID/PW)**. OBS도 URL에 자격증명 포함 가능.
3. **장애 시 화면 동작**: **마지막 정상 데이터 유지 + 작은 경고**.
   - API 실패 시 화면 데이터는 마지막 성공 시점으로 유지.
   - 하단 "실시간 상태" 패널의 점/문구만 경고색으로 변경 (시청자 거의 인지 못 함).
   - 큰 에러 배너/모달은 절대 노출 X (송출 안전).
4. **타임존**: **JST (일본시간)** 기준 표시.
   - 데이터 출처가 일본 채널이므로 일본 기준 시각 사용.
   - 표시 형식 예: `2025.05.24 14:30:00 (JST)`.
5. **Git 원격**: **GitHub (private repo)**.
6. **송출 해상도**: **1920×1080 (Full HD)** 기준 디자인 핏.
7. **도메인**: 사용 안 함. **Lightsail 고정 IP로 접속**. (HTTPS는 자체 서명 또는 HTTP 운영자 페이지 — PRD에서 결정)
8. **YouTube API 쿼터**: 150개 채널 × 4회/일 = 일 600 호출. `channels.list` 50개 묶음으로 일 12회. 무료 쿼터(10,000 units/day) 안에 충분 — 구현 시 검증만 하면 됨 (별도 결정 불필요).
